# Project Overview

> Status: Draft

## 1. Purpose

This repository holds reusable RE schema files and the first project-level scaffold for a Stage 1 / Stage 2 reverse-engineering workflow.

## 2. Current Goals

- Keep Stage 1 evidence-first and read-only.
- Keep Stage 2 entry-controlled and action-oriented.
- Standardize the document tree before adding project-specific content.

## 3. Primary Artifacts

- Risk register
- Open questions inside the risk register
- Trace plan
- E2E trace matrix
- Coupling map
- Clock and timing map
- Stage 2 risk resolution plan

## 4. Next Step

Populate the `docs/reference/` tree with project-specific findings once a target firmware project is selected.
